# 比赛更换仿真内容时一般需要修改什么

如果比赛使用仿真环境，对方通常会提供新的 Gazebo 场景。更换场景时，一般不需要一上来就改代码，优先检查和修改下面几类内容。

## 1. Gazebo 场景文件

Gazebo 使用 `.world` 文件描述仿真场景，例如墙、障碍物、地面、光照等。

更换比赛场景时，首先要确认新的 `.world` 文件已经放到功能包的 `worlds/` 目录中。

启动时需要把 `world:=` 改成新的场景文件名：

```bash
ros2 launch module_c_nav_teaching 01_gazebo_world.launch.py world:=新场景.world x_pose:=0.0 y_pose:=0.0
```

## 2. Nav2 地图文件

Nav2 不能直接拿 Gazebo 的 `.world` 文件导航，它需要二维地图文件，通常是一组：

```text
xxx.yaml
xxx.pgm 或 xxx.png
```

如果比赛提供了地图文件，需要放到功能包的 `maps/` 目录中。

启动 Nav2 时，把 `map:=` 改成对应的地图：

```bash
ros2 launch module_c_nav_teaching 02_nav2_known_map.launch.py map:=新地图.yaml
```

如果比赛只提供 Gazebo 场景，没有提供地图，就需要先启动仿真，再用 SLAM 建图，最后保存地图。

## 3. 机器人出生点

更换场景后，机器人出生点也可能需要调整。

Gazebo 启动时的出生点由下面两个参数决定：

```text
x_pose
y_pose
```

示例：

```bash
ros2 launch module_c_nav_teaching 01_gazebo_world.launch.py world:=新场景.world x_pose:=1.0 y_pose:=0.5
```

出生点要放在地图空闲区域，不能在墙里、障碍物上，或者离障碍物太近。

## 4. AMCL 初始位姿

Nav2 使用 AMCL 定位时，还需要发布初始位姿。

这个初始位姿要和 Gazebo 中机器人的出生点基本一致：

```text
Gazebo 出生点：x_pose, y_pose
AMCL 初始位姿：/initialpose 里的 x, y
```

如果两者不一致，RViz 中机器人位置可能会偏移，导航也可能失败。

## 5. 可能需要调整的 Nav2 参数

如果场景很窄、障碍物很多，或者机器人经常规划失败，可能需要调整 Nav2 参数文件。

常见参数文件：

```text
module_c_nav_teaching/config/nav2_params_module_c.yaml
```

常见需要关注的参数：

```text
robot_radius
inflation_radius
cost_scaling_factor
max_vel_x
max_speed_xy
planner tolerance
```

例如窄门场景中，如果 `inflation_radius` 太大，机器人可能会认为门太窄，导致无法规划通过。
